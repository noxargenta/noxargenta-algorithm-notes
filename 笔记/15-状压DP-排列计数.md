# 状压 DP — 排列计数（满足偏序关系）

来源：[Lazy_Shuffling.cpp](../competition/niukemanyschool2/Lazy_Shuffling.cpp) — 2025牛客多校第二场 L

## 问题模型

给定一个排列的某种约束关系（如逆序对的先后限制），统计满足所有约束的排列个数。

## 核心思想

用 `dp[s]` 表示 **已经放置在末尾的元素集合为 s** 时，满足约束的方案数。从空集开始，每次尝试往末尾添加一个元素，检查其所有前置条件是否已满足。

## 代码模板

```cpp
dp[0] = 1;
for (int s = 1; s < (1 << n); s++) {
    int sum = 0;
    for (int u = 0; u < n; u++) {
        if (s & (1 << u)) {
            if ((pre[u] & s) == 0) {      // u 的所有前置元素都已不在 s 中（已放置）
                sum += dp[s ^ (1 << u)];
            }
        }
    }
    dp[s] = sum;
}
// dp[(1<<n)-1] = 满足约束的排列数
```

**关键条件** `(pre[u] & s) == 0`：
- `pre[u]` 是必须放在 u **之前** 的元素集合（位掩码）
- `s` 是还未放置的元素集合
- 如果 `pre[u]` 中的元素还在 `s` 里（未放置），说明有前置约束未满足，不能放 u

## 例题：Lazy Shuffling

### 题意

给定一个排列 p，允许执行某种"懒人洗牌"操作。求所有可能的操作结果排列的数量。

### 推导

1. 对排列 p 统计逆序对，同时记录每个元素的"前驱"：`scin[p[j]] |= (1 << p[i])` 表示 p[i] 必须在 p[j] 之前（因为 p[i] > p[j] 且 i < j，是逆序对）
2. 若无逆序对（已有序），答案为 n!
3. 否则用状压 DP 统计满足前置约束的排列数 `dp[(1<<n)-1]`
4. 最终答案乘以 2（题目中洗牌过程的对称性）：**ans = 2 × dp[(1<<n)-1]**

### 代码

```cpp
void solve() {
    int n;
    cin >> n;
    vector<int> p(n);
    for (int i = 0; i < n; i++) {
        cin >> p[i];
        p[i]--;
    }

    vector<int> pre(n, 0);   // pre[u] = 必须在 u 之前的元素集合
    int inv = 0;
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (p[i] > p[j]) {
                inv++;
                pre[p[j]] |= (1 << p[i]);
            }
        }
    }

    if (inv == 0) {
        int ans = 1;
        for (int i = 1; i <= n; i++) ans = ans * i % MOD;
        cout << ans << endl;
        return;
    }

    vector<int> dp(1 << n, 0);
    dp[0] = 1;
    for (int s = 1; s < (1 << n); s++) {
        int sum = 0;
        for (int u = 0; u < n; u++) {
            if (s & (1 << u)) {
                if ((pre[u] & s) == 0) {    // u 的前置条件已满足
                    sum = (sum + dp[s ^ (1 << u)]) % MOD;
                }
            }
        }
        dp[s] = sum;
    }

    cout << 2LL * dp[(1 << n) - 1] % MOD << endl;
}
```

### 复杂度

- 时间 O(n × 2ⁿ)，n ≤ 20 左右可用
- 空间 O(2ⁿ)

## 经典应用

| 问题 | 做法 |
|------|------|
| 统计拓扑排序方案数 | 对 DAG 的边建立 pre[u]，跑状压 DP |
| 有偏序的排列计数 | 逆序对 / 竞赛中的大小关系约束 |
| Hamiltonian 路径计数 | `dp[s][v]` 表示已走集合 s，当前在 v |

## 注意事项

- n 不能太大（≤ 20 比较安全），否则 2ⁿ 爆炸
- `pre[u]` 的构建要根据具体约束的语义，保证偏序关系的**传递性**
- DP 顺序从 1 到 (1<<n)-1，保证子集 s ^ (1<<u) 已经在之前计算过
- 模数通常取 998244353 或 1e9+7
