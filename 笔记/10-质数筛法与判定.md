# 质数筛法与判定

来源：[P_1036_NOIP_2002_普及组_选数](../study/P_1036_NOIP_2002_普及组_选数.cpp)

## O(√n) 试除判定

单个数判质数，循环到 `i*i <= x` 即可。

```cpp
bool isPrime(int x) {
    if (x < 2) return 0;
    for (int i = 2; i * i <= x; i++) {
        if (x % i == 0) return 0;
    }
    return 1;
}
```

注意 `i*i` 可能爆 int，用 `i64` 更安全：
```cpp
bool isPrime(i64 x) {
    if (x < 2) return 0;
    for (i64 i = 2; i * i <= x; i++) {
        if (x % i == 0) return 0;
    }
    return 1;
}
```

## 欧拉线性筛 O(N)

用 `bitset<N>` 当作 bool 数组标记合数，只被最小质因子筛掉，每个合数只被筛一次。

```cpp
const int N = 100000008;
bitset<N> c;          // c[i] == 1 表示 i 是质数

c.set();              // 初始全标 1
c[0] = c[1] = 0;

vector<int> primes;
for (int i = 2; i < N; i++) {
    if (c[i]) primes.push_back(i);
    for (auto x : primes) {
        if (x * i >= N) break;
        c[x * i] = 0;
        if (i % x == 0) break;    // 关键：保证 x 是合数 i*x 的最小质因子
    }
}
```

之后判断质数直接用 `c[sum]` O(1) 查表，比每次 `isPrime(sum)` 快很多。

### 核心：`if (i % x == 0) break`

- `i` 是 `x` 的倍数时，`x` 已经是 `i` 的最小质因子
- 更大的质数 `x'` 筛 `i*x'` 时，`x` 才是真正的最小质因子（因为 `x < x'` 且 `x | i`）
- 加这行 break 保证每个合数只被最小质因子筛一次，复杂度 O(N)

### bitset vs bool 数组

`bitset<N>` 每元素占 1 bit，bool 数组占 1 byte（8 bit）。N=1e8 时 bitset 只用 ~12MB，bool 数组 ~100MB。质数筛常用 bitset 省内存。

## 埃氏筛 O(N log log N)

简单版：把每个质数的倍数全标掉。

```cpp
bitset<N> is_prime;
is_prime.set();
is_prime[0] = is_prime[1] = 0;
for (int i = 2; i * i <= N; i++) {
    if (is_prime[i]) {
        for (int j = i * i; j <= N; j += i) {
            is_prime[j] = 0;
        }
    }
}
```

效率比欧拉筛略差但代码更短，N 不大时足够用。

## 选择

| 方法 | 单次判定 | 多次判定 | 内存 |
|------|---------|---------|------|
| `isPrime()` 试除 | O(√n) | O(Q√n)，Q 大时炸 | O(1) |
| 埃氏筛 | O(N log log N) 预处理 | O(1) 查表 | ~12MB(bitset) |
| 欧拉线性筛 | O(N) 预处理 | O(1) 查表 | ~12MB(bitset) |

结论：多次判质数先跑筛，单次用试除。
