# 贡献法与乘法原理：子串分值和

问题是给一个字符串 s，求所有子串中不同字符个数的总和。

暴力 O(n^2) 枚举所有子串肯定不行。这类统计问题常用**贡献法**——不关注子串本身，而是算每个字符在多少个子串里算数。

关键：每个字符只在它**第一次出现**的位置贡献。同一个字符在后面再出现，前面的子串已经统计过它了，不能重复算。

这样就把问题转成了：对于位置 i 的字符，它能作为"第一次出现的字符"参与到哪些子串中？

左边界可以选 `(last[c], i]` 之间的任意位置，共 `i - last[c]` 种。
右边界可以选 `[i, n)` 之间的任意位置，共 `n - i` 种。
两边独立，乘起来就是该字符贡献的子串数。

```
sum += (i - last[c]) * (n - i)
```

## 实现

字符串前面加个空格变 1-indexed，避免处理边界时减出负数。last[26] 记录每个字符上次出现的位置，初始 0。

```cpp
void solve() {
    string s;
    cin >> s;
    s = " " + s;
    int last[26] = {0};
    int sum = 0;
    for (int i = 1; i < s.size(); i++) {
        int c = s[i] - 'a';
        sum += (i - last[c]) * (s.size() - i);
        last[c] = i;
    }
    cout << sum << endl;
}
```

O(n) 时间，O(1) 额外空间。

## 变体

贡献法 + 乘法原理的模式很多题都能用。比如 LeetCode 907（子数组最小值之和），对每个元素算它作为最小值的子数组个数，也是左右边界独立选，乘起来就是贡献。

核心就一句话：**不要枚举集合，转而算每个元素在哪些集合里是"特殊的那个"**，左右边界独立选 → 乘法原理直接算。
